Trend Project

Posted on 2019-08-18 | Post modified: 2019-09-01 | In Project

Trend project is a quantitative trading strategy that use machine learning method to analyze data in a trend to give prediction to the future.

Key factor that the model takes into consideration:

  • open, close, high, low
  • percentage change for previous day’s open and today’s open price.
  • Turnover rate

Why do I need to list factors with so such redundant?

Because the decision tree model is very stupid and can’t identify linear combination of exist factors. This can be a good thing(so all the combination is under my control, the only thing the machine can learn is the coefficient of the factor while I control all of the terms)

Model Development

Update Todo

  • The way that I set up strong trend and weak trend used future information.
  • How the industry optimal parameter used future information.

Version Updates

V0.0: Create the basic model that can determine the trend of asset.

  • Use merge method to get a approximation of the trend.
  • Run a for loop through 5~15 days length trend
  • Create Decision tree based on close price time series

V0.1: Add in more parameter on trend determination and find the best parameter for each industry.

  • Add in minimum change as a hard requirement for a trend
  • Set two minimum change so that there will be a strong trend and a weak trend
  • Find each industry’s optimal parameter
    • designed a method to test if the parameter fits the industry

V0.2: Improve the decision tree training

  • Improve the decision tree training parameters
  • Add in more factors:
    • Turnover rate
    • All the derived factors since decision tree can’t deal with anything else but linear coefficient of factors
  • Add in multi-class decision tree since in real trading it is completely different if the return is different

V0.3: Improve the mutli-class decision tree

  • Since the multi-class decision tree model can identify different level of return, then I add in score(weight) to judge the prediction result.

V0.4: Create back test system for the model

  • Create a day level back test system
  • The strategy is acting daily and the result is too good to be true

V1.0: Finally figures the problem in the model and changed it

  • The trimming action in the trend determination has a lot of problem
  • Correct the splitting and dividend problem that cause discontinuity in stock price

V2.0: Add in rolling calculation model:

  • Rolling calculation make the calculation much slower, but the effect has a significant change
  • Should always do rolling in machine learning

V2.1: Add in more advance machine learning algorithm to the model:

  • SVM is a great model since it can automatically study the linear relationship among different factors. But it is extremely time consuming to train and will add in some distraction terms to the model.
  • Random forest algorithm is an enhanced decision tree. The cost of training is acceptable and the improvement is significant. So currently this is the main algorithm I will continue to use

v2.21: Add in more factors:

  • Add in more fundamental factors: open, low, high
  • Add in more derived factors

V2.22: Improve the result of random forest

  • Since the random forest model will give a confidence level on how sure it is on the result, I can improve the model by only selecting results that random forest is more certain.

V2.3: Combined last two version and made some changes for efficiency purpose.

  • Changed to create a new model every 250 days for efficiency reason. The average correction rate is around 75% for binary prediction.
  • Create a new back test
  • correct the give signal method

V2.4: Correct many errors. The code finally works as expected. Still more to complete in next version
